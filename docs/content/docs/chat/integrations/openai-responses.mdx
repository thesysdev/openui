---
title: OpenAI Responses API
description: Connect OpenUI to a backend using OpenAI's Responses API streaming format.
---

Use `openAIResponsesAdapter` when your backend streams responses using `openai.responses.create()`.

If your backend also stores and loads messages in the Responses API's conversation items format, add `openAIConversationMessageFormat` as well.

### Stream only

If you only need to parse the streaming response (messages are stored in OpenUI's default format):

### With conversation message format

If your backend also stores messages in the Responses API's items list format and your thread endpoints return them in that shape:

**`openAIResponsesAdapter()`** parses the streaming response from OpenAI's Responses API. This API uses a different event format than chat completions — it streams server-sent events with types like `response.output_item.added` and `response.content_part.delta`. The adapter handles all of these and converts them into OpenUI's internal stream events.

**`openAIConversationMessageFormat`** is only needed if your backend stores and loads messages in the Responses API's items list format (`{ type: "message", role, content: [{ type: "input_text", text }] }`). It converts between this format and OpenUI's internal format when sending chat requests and loading saved threads.
