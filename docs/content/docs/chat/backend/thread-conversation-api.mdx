description: Build the backend API for creating, loading, and managing chat threads and messages.

By default, conversations live in memory and are lost on page reload. To let users save and resume conversations, your chat component needs a backend API for managing threads and messages.

This guide walks you through building that API step by step using Next.js.

## The API contract

OpenUI expects five RESTful endpoints. When you pass `threadApiUrl="/api/threads"` to a chat component, it calls these automatically:

| Action        | Method   | URL                         | Request body       | Response                                  |
| ------------- | -------- | --------------------------- | ------------------ | ----------------------------------------- |
| List threads  | `GET`    | `/api/threads`              | —                  | `{ threads: Thread[], nextCursor?: any }` |
| Create thread | `POST`   | `/api/threads`              | `{ firstMessage }` | `Thread`                                  |
| Delete thread | `DELETE` | `/api/threads/:id`          | —                  | —                                         |
| Update thread | `PUT`    | `/api/threads/:id`          | `Thread`           | `Thread`                                  |
| Load messages | `GET`    | `/api/threads/:id/messages` | —                  | `Message[]`                               |

Every `Thread` object must have this shape:

```typescript
type Thread = {
  id: string;
  title: string;
  createdAt: string | number;
};
```

## Choosing a message format

<Callout type="warn">**Important:** Every message must have a unique `id` field.</Callout>

Your backend can store messages in any format — OpenAI's `{ id, role, content }` objects, the OpenAI Responses API items list, or your own custom shape. The `messageFormat` prop on the frontend acts as a translator between your backend's format and OpenUI's internal format.

OpenUI ships with built-in converters for common formats:

| Converter                         | Your backend stores messages as...                   |
| --------------------------------- | ---------------------------------------------------- |
| Default (AG-UI)                   | OpenUI's native format: `{ id, role, content, ... }` |
| `openAIMessageFormat`             | OpenAI chat completions: `{ id, role, content }`     |
| `openAIConversationMessageFormat` | OpenAI Responses API items list with id              |

If your backend uses a different shape, you can write a custom converter for Ag-UI:

```typescript
const myCustomFormat: MessageFormat = {
  toApi(messages) {
    // Convert OpenUI messages → your backend's format (for chat requests)
    return messages.map((m) => ({ speaker: m.role, text: m.content }));
  },
  fromApi(messages) {
    // Convert your backend's format → OpenUI messages (when loading threads)
    return messages.map((m) => ({ id: m.id, role: m.speaker, content: m.text }));
  },
};
```

The examples below use `openAIMessageFormat`, so the message endpoints return and accept OpenAI's `{ role, content, id }` shape.

## Step-by-step implementation (Next.js)

### 1. Thread endpoints — `app/api/threads/route.ts`

This file handles listing and creating threads:

```typescript
import { db } from "@/lib/db";

// GET /api/threads — List all threads
export async function GET() {
  const threads = await db.threads.findMany({
    orderBy: { createdAt: "desc" },
  });

  return Response.json({
    threads: threads.map((t) => ({
      id: t.id,
      title: t.title,
      createdAt: t.createdAt,
    })),
  });
}

// POST /api/threads — Create a new thread
export async function POST(req: Request) {
  const { firstMessage } = await req.json();

  const thread = await db.threads.create({
    data: {
      title: firstMessage.content?.slice(0, 50) || "New conversation",
    },
  });

  return Response.json({
    id: thread.id,
    title: thread.title,
    createdAt: thread.createdAt,
  });
}
```

### 2. Single-thread endpoints — `app/api/threads/[id]/route.ts`

This file handles deleting and updating a specific thread:

```typescript
import { db } from "@/lib/db";

// DELETE /api/threads/:id — Delete a thread
export async function DELETE(req: Request, { params }: { params: { id: string } }) {
  await db.threads.delete({ where: { id: params.id } });
  return new Response(null, { status: 204 });
}

// PUT /api/threads/:id — Update a thread (e.g. rename)
export async function PUT(req: Request, { params }: { params: { id: string } }) {
  const body = await req.json();

  const thread = await db.threads.update({
    where: { id: params.id },
    data: { title: body.title },
  });

  return Response.json({
    id: thread.id,
    title: thread.title,
    createdAt: thread.createdAt,
  });
}
```

### 3. Message endpoint — `app/api/threads/[id]/messages/route.ts`

This endpoint returns the message history for a thread. The shape of each message must match the `messageFormat` you set on the frontend.

Since we're using `openAIMessageFormat`, we return messages as `{ role, content, id }` objects:

```typescript
import { db } from "@/lib/db";

// GET /api/threads/:id/messages — Load message history
export async function GET(req: Request, { params }: { params: { id: string } }) {
  const messages = await db.messages.findMany({
    where: { threadId: params.id },
    orderBy: { createdAt: "asc" },
  });

  return Response.json(messages);
}
```

## Connecting the frontend

Now that the backend is ready, point your chat component to it using `threadApiUrl`. Set `messageFormat` to match the format your endpoints return — in this case, `openAIMessageFormat`:

The two key props:

- **`threadApiUrl`** — points to the REST API you just built
- **`messageFormat`** — tells OpenUI how to translate messages to/from your backend's format
