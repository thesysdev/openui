Your backend needs a single endpoint that receives the conversation history and streams back a response from an LLM. This guide shows how to build one using the OpenAI SDK and Next.js.

## Next.js Route Handler

Create a route handler at `app/api/chat/route.ts`:

```typescript
import OpenAI from "openai";

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

export async function POST(req: Request) {
  const { messages } = await req.json();

  const stream = await openai.chat.completions.create({
    model: "gpt-4o",
    messages,
    stream: true,
  });

  const encoder = new TextEncoder();
  const readable = new ReadableStream({
    async start(controller) {
      for await (const chunk of stream) {
        const text = JSON.stringify(chunk) + "\n";
        controller.enqueue(encoder.encode(text));
      }
      controller.close();
    },
  });

  return new Response(readable, {
    headers: { "Content-Type": "text/event-stream" },
  });
}
```

This endpoint:

1. Reads the `messages` array from the request body
2. Calls OpenAI's chat completions API with streaming enabled
3. Forwards each chunk as newline-delimited JSON back to the client

## What `streamProtocol` and `messageFormat` do

These two props handle the translation between your backend and OpenUI's internal format:

- **`streamProtocol`** — parses the raw streaming response from your endpoint into OpenUI events. Different LLM providers stream in different formats.
- **`messageFormat`** — converts messages between OpenUI's internal format and the format your backend expects in the request body (and returns in loaded threads).

| Backend format          | `streamProtocol`           | `messageFormat`                   |
| ----------------------- | -------------------------- | --------------------------------- |
| AG-UI (default)         | Not needed                 | Not needed                        |
| OpenAI Chat Completions | `openAIAdapter()`          | `openAIMessageFormat`             |
| OpenAI Responses API    | `openAIResponsesAdapter()` | `openAIConversationMessageFormat` |

If your backend uses AG-UI, you don't need to set either prop — they default to AG-UI.

## Request format

When a user sends a message, OpenUI makes a `POST` request to your `apiUrl` with this body:

```json
{
  "messages": [
    { "role": "user", "content": "Hello!" },
    { "role": "assistant", "content": "Hi there! How can I help?" },
    { "role": "user", "content": "What's the weather like?" }
  ]
}
```

The exact shape of each message depends on your `messageFormat`. With `openAIMessageFormat`, messages use OpenAI's `{ role, content, id}` shape. By default, OpenUI uses [AG-UI](https://docs.ag-ui.com/concepts/messages#message-types) format..
