## Custom composer

Use the `composer` prop to replace the built-in message input with your own component. OpenUI passes your component everything it needs — send/cancel callbacks and streaming state — so you can focus on the UI:

<Tabs groupId="layout" items={['Copilot', 'FullScreen', 'BottomTray']} persist>
  <Tab value="Copilot">
    ```tsx
    import { Copilot } from "@openuidev/react-ui";
    import { useState } from "react";

    function MyComposer({ onSend, onCancel, isRunning }) {
      const [text, setText] = useState("");

      return (
        <form
          onSubmit={(e) => {
            e.preventDefault();
            if (text.trim() && !isRunning) {
              onSend(text);
              setText("");
            }
          }}
        >
          <input
            value={text}
            onChange={(e) => setText(e.target.value)}
            placeholder="Ask anything..."
            disabled={isRunning}
          />
          {isRunning ? (
            <button type="button" onClick={onCancel}>Stop</button>
          ) : (
            <button type="submit">Send</button>
          )}
        </form>
      );
    }

    function App() {
      return (
        <Copilot
          apiUrl="/api/chat"
          agentName="My Agent"
          composer={MyComposer}
        />
      );
    }
    ```

  </Tab>
  <Tab value="FullScreen">

### `ComposerProps`

OpenUI passes these props to your custom composer component:

| Prop                | Type                        | Description                                        |
| ------------------- | --------------------------- | -------------------------------------------------- |
| `onSend`            | `(message: string) => void` | Call this with the user's input to send a message  |
| `onCancel`          | `() => void`                | Cancel the in-progress response                    |
| `isRunning`         | `boolean`                   | `true` while the assistant is streaming a response |
| `isLoadingMessages` | `boolean`                   | `true` while conversation history is being loaded  |

```typescript
type ComposerProps = {
  onSend: (message: string) => void;
  onCancel: () => void;
  isRunning: boolean;
  isLoadingMessages: boolean;
};
```

Your component manages its own input state (the text value). When the user submits, call `onSend(text)` and clear your input. OpenUI handles everything else — adding the message to the thread, calling the backend, and streaming the response.

## Custom assistant messages

By default, OpenUI renders assistant messages as [GenUI](/docs/chat/genui) components. Use the `assistantMessage` prop to replace this with your own component:

<Tabs groupId="layout" items={['Copilot', 'FullScreen', 'BottomTray']} persist>
  <Tab value="Copilot">
    ```tsx
    import { Copilot } from "@openuidev/react-ui";

    function App() {
      return (
        <Copilot
          apiUrl="/api/chat"
          agentName="My Agent"
          assistantMessage={({ message }) => (
            <div className="my-assistant-bubble">
              <strong>Agent:</strong>
              <p>{message.content}</p>
            </div>
          )}
        />
      );
    }
    ```

  </Tab>
  <Tab value="FullScreen">
    ```tsx
    import { FullScreen } from "@openuidev/react-ui";

      return (
        <FullScreen

            <div className="my-assistant-bubble">

              <p>{message.content}</p>
            </div>
          )}
        />
      );
    }
    ```

  </Tab>
  <Tab value="BottomTray">
    ```tsx

    function App() {
      return (
    ```

  </Tab>

Your component receives the full `AssistantMessage` object:

```typescript
type AssistantMessage = {
  id: string;
  role: "assistant";
  content?: string;
  toolCalls?: ToolCall[];
  status?: "streaming" | "complete" | "awaiting_input";
  context?: JSONValue[];
};
```

You can use the `status` field to show a typing indicator while the response is still streaming:

```tsx
assistantMessage={({ message }) => (
  <div className="my-assistant-bubble">
    <p>{message.content}</p>
    {message.status === "streaming" && <span className="typing-dot">●</span>}
  </div>
)}
```

## Custom user messages

By default, OpenUI renders user messages as plain text bubbles. Use the `userMessage` prop to replace this with your own component:

<Tabs groupId="layout" items={['Copilot', 'FullScreen', 'BottomTray']} persist>
  <Tab value="Copilot">
    ```tsx
    import { Copilot } from "@openuidev/react-ui";

    function App() {
      return (
        <Copilot
          apiUrl="/api/chat"
          agentName="My Agent"
          userMessage={({ message }) => (
            <div className="my-user-bubble">
              {typeof message.content === "string"
                ? message.content
                : message.content.map((part, i) =>
                    part.type === "text" ? <span key={i}>{part.text}</span> : null
                  )}
            </div>
          )}
        />
      );
    }
    ```

  </Tab>

Your component receives the full `UserMessage` object:

```typescript
type UserMessage = {
  id: string;
  role: "user";
  content: string | InputContent[];
  context?: JSONValue[];
};

type InputContent =
  | { type: "text"; text: string }
  | { type: "binary"; mimeType: string; url?: string; data?: string; filename?: string };
```

User message content can be either a plain string or an array of `InputContent` parts (for multimodal inputs like text + images). Handle both cases in your component.
