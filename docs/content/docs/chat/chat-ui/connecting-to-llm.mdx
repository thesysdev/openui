title: Connecting to LLM
description: Point your chat components to a backend that streams LLM responses.
---

Every chat component needs to know where to send messages. You have two options: a simple URL (`apiUrl`) for standard use cases, or a custom function (`processMessage`) for full control.

## Using `apiUrl`

The simplest way is to pass the URL of your backend endpoint:

<Tabs groupId="layout" items={['Copilot', 'FullScreen', 'BottomTray']} persist>
  <Tab value="Copilot">
    ```tsx
    import { Copilot } from "@openuidev/react-ui";

    function App() {
      return <Copilot apiUrl="/api/chat" agentName="My Agent" />;
    }
    ```

  </Tab>
  <Tab value="FullScreen">
    ```tsx
    import { FullScreen } from "@openuidev/react-ui";

    function App() {
      return <FullScreen apiUrl="/api/chat" agentName="My Agent" />;
    }
    ```

  </Tab>
  <Tab value="BottomTray">
    ```tsx
    import { BottomTray } from "@openuidev/react-ui";

    function App() {
      return <BottomTray apiUrl="/api/chat" agentName="My Agent" />;
    }
    ```

  </Tab>
</Tabs>

When a user sends a message, OpenUI makes a `POST` request to your endpoint with the conversation history. Your endpoint processes the messages through an LLM and streams the response back.

See [Backend Implementation](/docs/chat/backend/nextjs) for how to build this endpoint.

## Using `processMessage`

If you need to customize the request â€” add **Authorization headers**, change the URL structure (e.g., path parameters), or transform the payload â€” use `processMessage` instead of `apiUrl`:

<Tabs groupId="layout" items={["Copilot", "FullScreen", "BottomTray"]} persist>
  <Tab value="Copilot">
    ```tsx
    <Copilot
      processMessage={async ({ threadId, messages, abortController }) => {
        // Example: Fetching with a Bearer token and dynamic URL
        return fetch(`/api/threads/${threadId}/chat`, {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${getToken()}`,
          },
          body: JSON.stringify({ messages }),
          signal: abortController.signal,
        });
      }}
      agentName="My Agent"
    />
    ```
  </Tab>
  <Tab value="FullScreen">
    ```tsx
    <FullScreen
      processMessage={async ({ threadId, messages, abortController }) => {
        return fetch(`/api/threads/${threadId}/chat`, {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${getToken()}`,
          },
          body: JSON.stringify({ messages }),
          signal: abortController.signal,
        });
      }}
      agentName="My Agent"
    />
    ```
  </Tab>
  <Tab value="BottomTray">
    ```tsx
    <BottomTray
      processMessage={async ({ threadId, messages, abortController }) => {
        return fetch(`/api/threads/${threadId}/chat`, {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${getToken()}`,
          },
          body: JSON.stringify({ messages }),
          signal: abortController.signal,
        });
      }}
      agentName="My Agent"
    />
    ```
  </Tab>
</Tabs>

The function receives:

| Parameter             | Type              | Description                                                |
| :-------------------- | :---------------- | :--------------------------------------------------------- |
| **`threadId`**        | `string`          | The current conversation's ID (if persistence is enabled). |
| **`messages`**        | `Message[]`       | The full message history.                                  |
| **`abortController`** | `AbortController` | Signal to cancel the request if the user clicks "Stop".    |

It must return a standard `Response` object (the result of `fetch`).

## Streaming Format

By default, OpenUI uses its own **OpenUI Protocol**. If your backend streams in a different format (like raw OpenAI chunks), pass a `streamProtocol` adapter:

```tsx
import { openAIAdapter } from "@openuidev/react-headless";

<Copilot apiUrl="/api/chat" streamProtocol={openAIAdapter()} agentName="My Agent" />;
```

| Adapter                        | Use when...                                                        |
| :----------------------------- | :----------------------------------------------------------------- |
| **Default**                    | Backend uses the OpenUI Protocol (Standard).                       |
| **`openAIAdapter()`**          | Backend uses `openai.chat.completions.create` with `stream: true`. |
| **`openAIResponsesAdapter()`** | Backend uses `openai.responses.create` (Realtime API).             |

See [Backend & Integrations](/docs/chat/backend/providers) for full setup guides.

## Message Format

By default, OpenUI uses its internal message structure. If your backend expects messages in a different shape (e.g., strictly OpenAI format), pass a `messageFormat` converter:

```tsx
import { openAIMessageFormat } from "@openuidev/react-headless";

<Copilot apiUrl="/api/chat" messageFormat={openAIMessageFormat} agentName="My Agent" />;
```

| Format                                | Use when...                                                |
| :------------------------------------ | :--------------------------------------------------------- |
| **Default**                           | Backend expects OpenUI message format.                     |
| **`openAIMessageFormat`**             | Backend expects standard OpenAI `{role, content}` objects. |
| **`openAIConversationMessageFormat`** | Backend expects OpenAI Responses API items.                |
