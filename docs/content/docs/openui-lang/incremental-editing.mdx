---
title: Incremental Editing
description: Edit existing UIs efficiently вҖ” the LLM patches only what changed, the parser merges it.
---

When a user says "add a pie chart" to an existing dashboard, the LLM doesn't regenerate everything.
It outputs only the changed and new statements. The parser merges them into the existing code.

## How it works

```
 Before (4 statements):                After merge (5 statements):

 root = Stack([header, tbl])           root = Stack([header, chart, tbl])  вҶҗ replaced
 header = CardHeader("Tickets")        header = CardHeader("Tickets")      вҶҗ kept
 tickets = Query("list_tickets",...)   tickets = Query("list_tickets",...) вҶҗ kept
 tbl = Table([...])                    tbl = Table([...])                  вҶҗ kept
                                       chart = PieChart(...)               вҶҗ added

 LLM only output 2 lines (root + chart). Parser merged by name.
```

**Initial generation** вҖ” LLM outputs full code:

```text
root = Stack([header, tbl])
header = CardHeader("Tickets")
tickets = Query("list_tickets", {}, {rows: []})
tbl = Table([Col("Title", tickets.rows.title), Col("Status", tickets.rows.status)])
```

**User says:** _"add a pie chart showing ticket status breakdown"_

**LLM outputs only the patch:**

```text
root = Stack([header, chart, tbl])
chart = PieChart(["Open", "Closed"], [
  @Count(@Filter(tickets.rows, "status", "==", "open")),
  @Count(@Filter(tickets.rows, "status", "==", "closed"))
], "donut")
```

The parser merges by statement name:

- `root` is redefined вҶ’ new definition wins (now includes `chart`)
- `chart` is new вҶ’ added
- `header`, `tickets`, `tbl` вҶ’ kept from original (not in the patch, so unchanged)

## Merge rules

- **Same name** вҶ’ new definition replaces old
- **New name** вҶ’ added to the program
- **Missing from patch** вҶ’ kept from original (not deleted)
- **Explicit deletion** вҶ’ remove a statement from the `root` children list, it becomes unreachable and gets garbage-collected

## Why this matters

- **Fewer tokens** вҖ” LLM outputs 2-3 lines instead of regenerating 20+
- **Faster** вҖ” less to stream, less to parse
- **Preserves context** вҖ” existing queries, state, and bindings stay intact
- **Works with streaming** вҖ” each patched line renders as it arrives

## Enabling edit mode

Set `editMode: true` in your prompt config so the LLM knows it can output partial patches:

```ts
const config: PromptSpec = {
  ...componentSpec,
  editMode: true,
  inlineMode: true, // allows text + code responses
};
```

The `inlineMode` flag lets the LLM respond with explanation text alongside the code patch,
so it can say "I added a pie chart" before the code block.

See [System Prompts](/docs/openui-lang/system-prompts) for the full `PromptSpec` reference.
